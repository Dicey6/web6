Blockchain Verification

The backend should verify that:

* The payment arrived at the configured treasury wallet.
* The received amount matches the expected SOL amount (allowing a small configurable tolerance for rounding if necessary).
* The payment arrived before the order expired.
* The transaction has not already been processed.

After successful verification:

* Mark the order as Paid.
* Activate the purchased challenge.
* Trigger the existing referral logic.
* Update all related records.

Security

* Duplicate confirmations
* Replay attacks
* Processing the same transaction multiple times
* Double challenge activation
* Expired orders being activated

The same blockchain transaction must never activate more than one order.
