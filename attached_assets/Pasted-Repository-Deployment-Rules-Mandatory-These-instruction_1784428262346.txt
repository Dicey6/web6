Repository & Deployment Rules (Mandatory)

These instructions are mandatory.

Do not change the deployment strategy or repository structure.

⸻

Current Task

You are working only on the Website repository.

Do not commit, modify, or push changes to the Telegram bot repository.

If, during implementation, you discover that the Telegram bot requires changes to remain compatible with the website, do not implement those changes.

Instead:

* Clearly explain why the bot needs to change.
* Specify the exact files that need modification.
* Describe the required changes in detail.
* Explain why those changes are necessary.

I will implement those bot changes manually in the bot repository.

⸻

GitHub Workflow

Only the Website repository will receive commits and be pushed to GitHub.

When the backend work is complete:

* Commit all website changes.
* Push only the website repository to the new GitHub repository.

Do not attempt to clone, modify, commit, or push the Telegram bot repository.

The bot repository is maintained separately.

⸻
